# UpscaleAI — Project Status Reference

Last updated: 2026-04-14. Keep this file current when state changes so future sessions don't need to re-read every file.

## What this project is
FastAPI service that upscales images. Two backends:
1. **Real-ESRGAN** (AI-based, 4x, CPU) — optional; uses xinntao `realesrgan` 0.3.0.
2. **Pillow** (lanczos/bicubic/bilinear/nearest, 1–8x) — always works.

Canonical server: `upscale_api_unified.py`. Auto-detects the Real-ESRGAN backend at import time (`REALESRGAN_AVAILABLE`) and falls back to Pillow transparently. `/status` reports capability. Requesting `method=realesrgan` when the backend is unavailable returns HTTP 503.

## File map

### Source (keep)
- `upscale_api_unified.py` — **the** API. `GET /`, `GET /status`, `POST /upscale/?method=&scale=&model=`.
  - `initialize_upsampler()` is module-scope, cached per model in `_upsampler_cache`.
  - `file.content_type` is None-guarded.
  - `__main__` uses `uvicorn.run(app, ...)` without `reload=True` (reload breaks subprocess-based tests).
- `image_utils.py` — path helpers for `images/{input,output/{realesrgan,simple},temp}/`. Creates dirs on import.
- `test_unified_api.py` — assert-based E2E driver. Spawns server on port 8765, runs 18 checks, writes outputs, prints `PASS N/N`, exits non-zero on failure.
- `requirements.txt` — all deps active and pinned (incl. `realesrgan`, `torch`, `torchvision`, `opencv-python`, `python-multipart`, `httpx`).
- `README.md`, `LICENSE`, `.gitignore`.

## Known caveats / gotchas (capture these, they bite every new install)
1. **`basicsr` 1.4.2 vs `torchvision` ≥ 0.17** — `basicsr/data/degradations.py` imports `torchvision.transforms.functional_tensor`, which no longer exists. Patch the one line to import from `torchvision.transforms.functional`. The README documents this. If you recreate the venv, you'll need to re-patch.
2. **Real-ESRGAN requires Python ≤ 3.12.** `realesrgan==0.3.0` + `basicsr==1.4.2` don't install cleanly on 3.13.
3. **First Real-ESRGAN call downloads ~85 MB of weights** into the project root. Subsequent calls are cached locally and via `_upsampler_cache` in-process.
4. **Nothing is committed yet.** The unified API, image_utils, tests, and the `images/` tree are untracked. `upscale_api.py` still has the pre-unified diff staged. A cleanup commit is overdue.

## Suggested next steps
- Delete the legacy files listed above and commit.
- Consider pinning the basicsr patch as a `pip install --upgrade` post-install script or forking `basicsr` to avoid re-patching every fresh install.
- If a second machine will run this, document the `.pth` download step or pre-seed the weights.

## Quick commands
```bash
# Recreate venv (from scratch)
rm -rf venv && python3 -m venv venv
venv/bin/pip install -r requirements.txt
# Patch basicsr (until the maintainers fix or we switch forks)
sed -i '' 's/from torchvision.transforms.functional_tensor import/from torchvision.transforms.functional import/' \
  venv/lib/python3.11/site-packages/basicsr/data/degradations.py

# Run API
venv/bin/python upscale_api_unified.py          # binds 0.0.0.0:8000

# Run the full test harness
venv/bin/python test_unified_api.py             # expect PASS 18/18

# Manual smoke test
curl -s localhost:8000/status
curl -X POST "localhost:8000/upscale/?method=lanczos&scale=2" \
  -F "file=@images/input/test_image.png" --output /tmp/out.png
```
